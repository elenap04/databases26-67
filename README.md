## Τεχνολογίες

- **DBMS:** MySQL 8.0
- **Backend:** Node.js + Express (REST API, port 3000)
- **Frontend:** Vanilla HTML / CSS / JavaScript (port 8080)
- **Data loader:** Python 3

## Εγκατάσταση και Εκτέλεση

### Προαπαιτούμενα

**Windows & Linux:**
- MySQL 8.0
- Node.js 18+
- Python 3.10+

---

### Βήμα 1 - Δημιουργία Βάσης Δεδομένων

Συνδεθείτε στη MySQL και εκτελέστε το install script:

**Windows (MySQL Command Line Client):**
```sql
SOURCE C:/path/to/ygeiopolis/sql/install.sql;
```

**Linux:**
```bash
mysql -u root -p < sql/install.sql
```

---

### Βήμα 2 - Ρύθμιση Environment Variables

Δημιουργήστε τα παρακάτω `.env` αρχεία:

**`sql/.env`**
```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=yourpassword
DB_PORT=3306
DB_NAME=Ygeiopolis_db
DATA_DIR=./data
```

**`backend/.env`**
```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=yourpassword
DB_NAME=Ygeiopolis_db
DB_PORT=3306
PORT=3000
```

**`frontend/.env`**
```env
PORT=8080
API_URL=http://localhost:3000
```

---

### Βήμα 3 - Φόρτωση Δεδομένων

```bash
cd sql
pip install mysql-connector-python python-dotenv
python load_final.py --password yourpassword
```

---

### Βήμα 4 - Εκκίνηση Backend

```bash
cd backend
npm install
node server.js
```

Το API είναι διαθέσιμο στο `http://localhost:3000`.

---

### Βήμα 5 - Εκκίνηση Frontend

```bash
cd frontend
node server.js
```

Η εφαρμογή είναι διαθέσιμη στο `http://localhost:8080`.

## Παραδοχές Σχεδιασμού

| # | Κατηγορία | Παραδοχή |
|---|-----------|----------|
| 1 | Ασθενείς | Ένας ασθενής μπορεί να νοσηλευτεί για περισσότερες από μία ασθένειες κατά τη διάρκεια της ίδιας νοσηλείας (πολλαπλές διαγνώσεις εισαγωγής και εξόδου). |
| 2 | Κτίριο | Οι όροφοι του νοσοκομείου κυμαίνονται από -3 έως 20, όπως ορίζεται από CHECK constraint. |
| 3 | Προσωπικό | Οι βαθμίδες ιατρών (Resident, Junior Attending, Senior Attending, Director) και νοσηλευτών (Nursing Assistant, Nurse, Head Nurse) ορίζονται ως CHECK constraints. |
| 4 | Προσωπικό | Ειδικευόμενοι (Resident) έχουν υποχρεωτικά επόπτη. Διευθυντές (Director) δεν έχουν επόπτη. Απαγορεύεται κυκλική εποπτεία. |
| 5 | Προσωπικό | Κάθε ιατρός μπορεί να ανήκει σε ένα ή περισσότερα τμήματα. Νοσηλευτές και διοικητικοί ανήκουν ακριβώς σε ένα τμήμα. |
| 6 | Τμήματα | Ο Διευθυντής κάθε τμήματος πρέπει να είναι ιατρός βαθμίδας Director. Κανένας δεν μπορεί να διευθύνει περισσότερα από ένα τμήματα. |
| 7 | Email | Το format του email ελέγχεται από τη βάση μέσω LIKE (`%@%.%`). Ο έλεγχος ύπαρξης πρέπει να γίνεται από το application layer. |
| 8 | Αξιολόγηση | Η κλίμακα Likert αναπαρίσταται αριθμητικά με ακέραιους 1–5. Κάθε νοσηλεία αξιολογείται το πολύ μία φορά και μόνο μετά την έξοδο. |
| 9 | Νοσηλείες | Ο χρόνος νοσηλείας μετριέται σε ημέρες (διανυκτερεύσεις) |
| 10 | Νοσηλείες | Κάθε νοσηλεία συνδέεται υποχρεωτικά με μία triage_entry. Η ημερομηνία εισαγωγής πρέπει να είναι μεταγενέστερη της εξυπηρέτησης triage. |
| 11 | Κλίνες | Δεν επιτρέπεται ταυτόχρονη νοσηλεία δύο ασθενών στην ίδια κλίνη. Το status κλινών ενημερώνεται αυτόματα μέσω scheduled event κάθε 1 λεπτό. |
| 12 | Ημερομηνίες | Επιβάλλεται αυστηρότητα στις ημερομηνίες όλων των ιατρικών πράξεων για λόγους ιστορικού ασθενή και διαφάνειας. Υλοποιείται με triggers καθώς η MySQL δεν επιτρέπει `CURDATE()` σε CHECK constraints. |
| 13 | Εφημερίες | Κάθε βάρδια έχει ακριβώς καθορισμένα ωράρια: Πρωινή 07:00-15:00, Απογευματινή 15:00-23:00, Νυχτερινή 23:00-07:00. |
| 14 | Εφημερίες | Μεταξύ δύο βαρδιών του ίδιου ατόμου πρέπει να μεσολαβούν τουλάχιστον 8 ώρες ανάπαυσης. Μέγιστος αριθμός βαρδιών/μήνα: ιατροί 15, νοσηλευτές 20, διοικητικοί 25. |
| 15 | Εφημερίες | Απαγορεύονται περισσότερες από 3 συνεχόμενες νυχτερινές βάρδιες. Η οριστικοποίηση απαιτεί ≥3 ιατρούς, ≥6 νοσηλευτές, ≥2 διοικητικούς. |
| 16 | Χειρουργεία | Κάθε χειρουργείο έχει χρόνο έναρξης και **εκτιμώμενη** διάρκεια. Τα χειρουργεία μπορούν να εισαχθούν πριν γίνουν (προγραμματισμένα), κατά την έναρξη (επείγοντα) ή εκ των υστέρων. |
| 17 | Χειρουργεία | Το status των χειρουργείων ενημερώνεται αυτόματα μέσω scheduled event βάσει ώρας έναρξης και εκτιμώμενης ώρας λήξης. |
| 18 | Χειρουργεία | Τα χειρουργεία εισάγονται με `is_finalized = 0` (default). Η οριστικοποίηση απαιτεί σύνδεση με νοσηλεία και γίνεται μέσω ξεχωριστής ενέργειας. |
| 19 | Ιατρικές Πράξεις | Οι διαγνωστικές/θεραπευτικές πράξεις δεν έχουν αυστηρό χρονικό περιορισμό χώρου - π.χ. βιοψία ή αλλαγή επιδέσμου δεν απαιτεί αποκλειστική αίθουσα. |
| 20 | Συνταγογράφηση | Απαγορεύεται η συνταγογράφηση αν ο ασθενής έχει αλλεργία σε δραστική ουσία του φαρμάκου (trigger). Δεν επιτρέπεται τροποποίηση συνταγής - απαιτείται διαγραφή και νέα εισαγωγή. |
| 21 | Triage | Η εξυπηρέτηση γίνεται με FIFO ανά επίπεδο επείγοντος. Το triage είναι αποκλειστικά στο τμήμα Emergency Department. |
| 22 | ΚΕΝ | Το total_cost υπολογίζεται αυτόματα κατά την έξοδο (trigger BEFORE UPDATE): βασικό κόστος ΚΕΝ + ημερήσια χρέωση για ημέρες πέραν ΜΔΝ + χειρουργεία + εξετάσεις + πράξεις. |
| 23 | Q1 | Λόγω δυνατότητας πολλαπλής ασφάλισης, τα έσοδα νοσηλείας εμφανίζονται τόσες φορές όσοι και οι ασφαλιστικοί φορείς του ασθενή. |
| 24 | Q4 | Ως «ασθενείς του ιατρού» ορίζονται οι ασθενείς νοσηλειών όπου ο ιατρός συμμετείχε σε surgery, lab_exam ή med_proc (μέσω exam_doc). |
